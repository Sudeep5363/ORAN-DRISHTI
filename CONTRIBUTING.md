# Contributing to Oran-Drishti

Thank you for considering contributing to **Oran-Drishti**! We welcome contributions that help improve sacred grove protection and ecological monitoring.

## Code of Conduct

- Be respectful and inclusive
- Focus on the mission: environmental conservation
- Provide constructive feedback
- Respect others' ideas and work

## How to Contribute

### 1. **Report Issues**
- Check existing issues first
- Provide clear title and description
- Include steps to reproduce, environment details, and screenshots
- Label issues appropriately (bug, feature, documentation)

### 2. **Submit Pull Requests**
- Fork the repository
- Create a feature branch: `git checkout -b feature/amazing-feature`
- Make focused, atomic commits with clear messages
- Ensure code passes linting: `npm run lint`
- Submit PR with clear description of changes

### 3. **Improve Documentation**
- Fix typos and unclear explanations in README/docs
- Add usage examples for new features
- Document new algorithms or data sources

### 4. **Suggest Features**
- Describe the use case and expected behavior
- Explain why this feature matters for the project
- Provide mockups or examples if applicable

## Development Setup

```bash
# Clone and navigate
git clone https://github.com/yourusername/ORAN-DRISHTI.git
cd ORAN-DRISHTI

# Install dependencies
npm install

# Start dev server
npm run dev

# Run type checking
npm run lint
```

## Coding Standards

- **TypeScript**: Use strict mode, proper typing
- **Components**: Functional components with hooks
- **Naming**: descriptive names (avoid abbreviations)
- **Comments**: Use comments for complex logic only
- **Testing**: Add tests for new features

## Commit Message Format

```
<type>: <subject>

<body>
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

Examples:
- `feat: add real-time alert notifications`
- `fix: correct harmonic analysis calculation errors`
- `docs: improve api documentation`

## Areas for Contribution

- 🎨 **Frontend**: UI/UX improvements, new visualization components
- 🧠 **ML/Algorithms**: Better anomaly detection, new validation methods
- 🌍 **Data Integration**: New satellite data sources, better STAC support
- 📊 **Analysis**: Statistical analysis improvements, accuracy metrics
- 📖 **Documentation**: Tutorials, API docs, technical guides
- 🧪 **Testing**: Unit tests, integration tests, edge cases

## Questions?

- Open a GitHub discussion
- Check existing documentation in `/docs`
- Contact maintainers via email
